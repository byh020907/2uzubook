# 개선해야 할 사항

* 동아리는 따로 분리하여 동아리 추가/수정/삭제 기능이 따로 존재하고 레주메는 동아리원들 모두에게 적용

* 시리얼 키가 짧아 무차별 대입 공격(Brute Force)에 당할 위험이 크므로 연속 5회 실패 시 대기시간 적용 등 방안 고려 (IP 이용)

* 상 수여 기관, 자격증 발급 기관 등은 직접 입력하게 하는 방식보다는 주어진 선택지에서 선택할 수 있게 변경 (모든 기관의 목록 필요)

* Programming 아래의 Java와 같이 서로 계층적인 관계에 있는 키워드는 조금 더 효율적으로 관리할 수 있는 방안 고려 (예를 들어, parent 열을 추가하여 Java만 포함시켜도 Programming 검색 시 찾아지도록 변경)
